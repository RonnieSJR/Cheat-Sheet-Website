HTML Cheat Sheet

Document Structure


<!DOCTYPE html>
<html lang="en">
<head>
    <meta charset="UTF-8">
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    <title>Document Title</title>
</head>
<body>
    <!-- Content goes here -->
</body>
</html>

This is the basic structure of an HTML document. It includes the DOCTYPE declaration, html, head, and body elements.

Use this structure as the starting point for every HTML document you create.

Text Elements

  • <h1> to <h6> Headings

    Define six levels of section headings in HTML. <h1> is the highest (most important) level and <h6> is the lowest.

    Use for structuring content and improving SEO. Maintain a logical hierarchy (don't skip levels).

  • <p> Paragraph

    Defines a paragraph of text.

    Use for blocks of text, such as paragraphs in articles, blog posts, or general content.

  • <strong> Strong importance

    Indicates strong importance, seriousness, or urgency for its contents.

    Use to emphasize important words or phrases within text. Typically rendered in bold.

  • <em> Emphasized text

    Marks text that has stress emphasis.

    Use for words that have a stressed emphasis compared to surrounding text. Typically rendered in italics.

  • <br> Line break

    Produces a line break in text.

    Use sparingly to create line breaks, such as in addresses or poetry. Avoid for spacing in layouts.

  • <hr> Horizontal rule

    Represents a thematic break between paragraph-level elements.

    Use to separate content sections or to indicate a change in topic within a section.

Links and Media

  • <a href="url"> Hyperlink

    Creates a hyperlink to web pages, files, email addresses, or anything else a URL can address.

    Use for navigation within a site, linking to external resources, or creating clickable text/images.

  • <img src="url" alt="description"> Image

    Embeds an image into the document.

    Use to display images in your content. Always include descriptive alt text for accessibility.

  • <video src="url"> Video

    Embeds a media player for video playback.

    Use to include video content in your web pages. Consider adding controls and poster attributes.

  • <audio src="url"> Audio

    Embeds sound content in documents.

    Use for audio playback on web pages, such as music or podcasts.

Lists

  • <ul> Unordered list

    Represents an unordered list of items.

    Use for lists where the order of items doesn't matter, typically rendered with bullet points.

  • <ol> Ordered list

    Represents an ordered list of items.

    Use for lists where the order of items is meaningful, typically rendered with numbers.

  • <li> List item

    Represents an item in a list.

    Use within <ul> or <ol> elements to define individual list items.

  • <dl> Description list

    Represents a description list.

    Use for a list of terms and their associated descriptions, such as a glossary.

  • <dt> Description term

    Specifies a term in a description list.

    Use within a <dl> to define the term being described.

  • <dd> Description details

    Provides the description or definition in a description list.

    Use within a <dl> to provide the description for the preceding <dt>.

Semantic Elements

  • <header> Header of a section or page

    Represents introductory content, typically a group of introductory or navigational aids.

    Use for a container of introductory content or a set of navigational links.

  • <nav> Navigation links

    Represents a section of a page that contains navigation links.

    Use for the main navigation menu of a website or for a table of contents.

  • <main> Main content of the page

    Represents the dominant content of the body of a document.

    Use to encapsulate the primary content of your web page. There should be only one <main> element per page.

  • <article> Self-contained content

    Represents a self-contained composition in a document, which is intended to be independently distributable or reusable.

    Use for blog posts, news articles, product cards, or any content that can stand alone.

  • <section> Thematic grouping of content

    Represents a standalone section of a document, which doesn't have a more specific semantic element to represent it.

    Use to divide content into sections, such as chapters, tabbed content, or themed groups of content.

  • <aside> Sidebar content

    Represents a portion of a document whose content is only indirectly related to the document's main content.

    Use for sidebars, call-out boxes, or any content that is tangentially related to the main content.

  • <footer> Footer of a section or page

    Represents a footer for its nearest sectioning content or sectioning root element.

    Use for footer content such as copyright information, related documents, or author bios.

Tables

  • <table> Table

    Represents tabular data.

    Use to display data in rows and columns. Avoid using tables for layout purposes.

  • <tr> Table row

    Defines a row of cells in a table.

    Use within a <table> to create a horizontal row of cells.

  • <th> Table header cell

    Defines a header cell in a table.

    Use for column or row headers to provide context for the data in the table.

  • <td> Table data cell

    Defines a cell of a table that contains data.

    Use for the actual data content within a table.

  • <thead> Table head

    Defines a set of rows defining the head of the columns of the table.

    Use to group header content in a table, typically the first row with column names.

  • <tbody> Table body

    Encapsulates a set of table rows, indicating that they comprise the body of the table.

    Use to group the main content of a table, separating it from headers and footers.

  • <tfoot> Table footer

    Defines a set of rows summarizing the columns of the table.

    Use for footer content in a table, such as totals or summary information.

Forms

  • <form> Form

    Represents a document section containing interactive controls for submitting information.

    Use to create a form for user input, such as login forms, search bars, or data submission forms.

  • <input> Input field

    Creates interactive controls for web-based forms to accept data from the user.

    Use for various types of user input, including text, passwords, numbers, checkboxes, and radio buttons.

  • <textarea> Multiline text input

    Represents a multi-line plain-text editing control.

    Use when you need to collect larger amounts of text from users, such as comments or messages.

  • <button> Clickable button

    Represents a clickable button.

    Use for form submissions, triggering JavaScript events, or any clickable action on a web page.

  • <select> Drop-down list

    Represents a control that provides a menu of options.

    Use when you want users to select one or multiple options from a list, such as choosing a country or category.

  • <option> Option in a select element

    Defines an option in a select element.

    Use within a <select> element to define the available choices for the user.

  • <label> Label for an input

    Represents a caption for an item in a user interface.

    Use to provide a text label for form controls, improving accessibility and usability.

  • <fieldset> Group of form controls

    Groups several controls as well as labels within a web form.

    Use to organize related form controls into logical groups, such as personal information or shipping details.

  • <legend> Caption for a fieldset

    Represents a caption for the content of its parent <fieldset>.

    Use to provide a title or explanatory caption for a <fieldset>.

Metadata and Document-level Elements

  • <meta> Metadata

    Defines metadata about an HTML document.

    Use to specify character encoding, page description, keywords, author, and other metadata.

  • <link> Link to external resource

    Specifies relationships between the current document and an external resource.

    Use to link to external stylesheets, define favicon, or establish relationships with other pages.

  • <style> Embedded CSS styles

    Contains style information for a document or part of a document.

    Use to define CSS styles directly within an HTML document, though external stylesheets are generally preferred.

  • <script> Embedded or external JavaScript

    Used to embed or reference executable code, typically JavaScript.

    Use to include JavaScript directly in the document or to reference external script files.

Text Formatting

  • <span> Inline container

    Generic inline container for phrasing content.

    Use to apply styles to a specific portion of text or to group inline elements for scripting purposes.

  • <pre> Preformatted text

    Represents preformatted text which is to be presented exactly as written in the HTML file.

    Use for displaying code blocks, ASCII art, or any text where whitespace and formatting should be preserved.

  • <code> Code snippet

    Displays its contents styled in a fashion intended to indicate that the text is a short fragment of computer code.

    Use for marking up inline code snippets or references to code elements like variable names or file names.

  • <blockquote> Block quotation

    Indicates that the enclosed text is an extended quotation.

    Use for long quotations that are set off from the main text, often with indentation.
